For this project, I built a multi-page .NET MAUI application consisting of a Home screen and a Student Profile page. I utilized VerticalStackLayout to organize UI elements like Labels, Images, and Buttons to meet the specific layout requirements. The most significant technical challenge I solved involved a CS0111 error caused by a naming conflict where my MainPage code-behind was incorrectly identified as a second ProfilePage class. I resolved this by manually synchronizing the x:Class attribute in the XAML files with the partial class definitions in the files to ensure the compiler could properly link the two. This fix was essential for allowing the OnTipClicked and OnGoToProfileClicked event handlers to be recognized by the project’s build system. I also configured the App.xaml.cs file to use a NavigationPage wrapper, which is required to enable the PushAsync and PopAsync stack operations between the two pages. To fulfill the asset requirements, I managed the Resources/Images folder to ensure the campus and mascot images were correctly bundled as MauiImage types. Overall, this assignment gave me a practical look at how .NET MAUI handles the relationship between stating UI design and its underlying logic.
